# LiftPDF Growth Roadmap

Date: 2026-07-10

Status: growth roadmap after product V1.

## Growth Principle

LiftPDF should grow by owning a differentiated promise:

> Private browser-based PDF tools.

The roadmap must be driven by Search Console, GA4 events and real usage. Do not create large volumes of content before the first query and funnel data exists.

## 30 Days

Goal: measurement and indexing foundation.

Target: first stable baseline, not aggressive traffic.

Expected outcome:

- Search Console verified.
- Sitemap submitted.
- GA4 Measurement ID configured.
- Tool funnel events visible.
- Top pages and queries tracked weekly.
- First 5-10 priority pages monitored.

Actions:

1. Add `NEXT_PUBLIC_GA_MEASUREMENT_ID` in Vercel.
2. Confirm GA4 receives page views and tool events.
3. Submit `https://liftpdf.com/sitemap.xml` in Search Console.
4. Inspect homepage, `/pdf-tools`, `/jpg-to-pdf`, `/merge-pdf`, `/protect-pdf`.
5. Start weekly SEO monitoring.
6. Start weekly CRO review from event funnels.
7. Publish only the highest-confidence guide pages if Search Console supports demand.

Traffic milestone:

- 100 visitors/day becomes realistic only after indexing and early long-tail impressions appear.

## 90 Days

Goal: build topical authority around current tools.

Target:

- 100 visitors/day consistently.
- Early query clusters for image-to-PDF, merge/split/compress, privacy PDF tools.

Actions:

1. Publish 10-20 high-quality guides from the editorial roadmap.
2. Create comparison pages only where the angle is honest: privacy/browser processing.
3. Improve titles and meta descriptions for pages with high impressions and low CTR.
4. Add internal links from guides to tools and related tools.
5. Monitor event drop-off per tool.
6. Prioritize bug fixes from `error_tool` data.
7. Build first backlink outreach list.

Traffic milestone:

- 1000 visitors/day is possible only if multiple pages start ranking in positions 3-10 for long-tail queries and if backlinks begin to land.

## 180 Days

Goal: become a recognizable alternative in privacy-first PDF workflows.

Target:

- 500-2000 visitors/day depending on indexing, content quality and backlinks.

Actions:

1. Expand content clusters only where Search Console proves impressions.
2. Refresh FAQs from real queries.
3. Create visual assets for Google Images on priority tools.
4. Add case-specific guide content for Windows, Mac, iPhone, Android, privacy and quality.
5. Decide Office conversion and OCR strategy from feasibility and demand.
6. Build 20-40 quality backlinks from directories, open-source lists, privacy/tool roundups and content partnerships.

Traffic milestone:

- 10000 visitors/day is unlikely without major content breadth, backlinks and high-demand missing tools such as PDF to Word or OCR.

## 365 Days

Goal: compete as a serious global PDF utility brand.

Target:

- 1000-10000 visitors/day depending on execution and authority.

Actions:

1. Build 50-100 excellent guides without cannibalization.
2. Strengthen comparison pages.
3. Add only high-demand tools that are technically honest.
4. Maintain performance and privacy advantage.
5. Build recurring backlink partnerships.
6. Track rankings and update pages quarterly.
7. Develop brand trust assets: security page, privacy architecture, browser-processing proof.

## KPI Targets

| Stage | KPI | Target |
|---|---|---|
| 30 days | Indexed priority pages | 90%+ |
| 30 days | GA4 funnel events | Receiving data |
| 90 days | Daily visitors | 100+ |
| 90 days | Tool completion rate | Baseline established |
| 180 days | Daily visitors | 500-2000 |
| 180 days | Backlinks | 20+ relevant links |
| 365 days | Daily visitors | 1000-10000 |
| 365 days | Content library | 50-100 strong pages |

## Guardrails

- Do not publish 100 pages at once.
- Do not create thin pages.
- Do not chase keywords that require OCR or Office conversion unless LiftPDF can satisfy the intent.
- Do not sacrifice browser privacy for growth.
- Do not add analytics that collects document names, passwords or content.
