# LiftPDF Editorial Roadmap

Date: 2026-07-08  
Scope: first 20 priority guide briefs.  
Objective: create detailed plans for the first editorial wave without writing the final guides.

## Priority Logic

The first guides should support existing LiftPDF tools and attack realistic long-tail openings rather than impossible head terms. The strongest angle is:

> complete PDF tasks privately in the browser without uploading files.

Priority is based on:

- existing tool availability;
- search intent strength;
- realistic ranking opportunity for a young domain;
- internal linking value;
- product trust value;
- conversion potential.

## Editorial Risk Controls

- Do not publish unavailable tool guides as if tools exist.
- Do not overuse "free" if future pricing changes might alter positioning.
- Do not claim OCR support.
- Do not claim advanced compression levels.
- Do not imply Unlock PDF can bypass passwords.
- Do not duplicate tool landing page copy.
- Do not publish thin guides under 800 words unless the query is extremely narrow.
