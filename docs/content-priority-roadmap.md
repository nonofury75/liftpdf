# LiftPDF Content Priority Roadmap

Date: 2026-07-08  
Scope: prioritization for future content clusters.  
Status: planning only. No content should be published from this roadmap without production review.

## 1. Priority Framework

Pages are ranked by:

- SEO potential;
- realistic difficulty for a young domain;
- conversion ROI;
- support for existing tools;
- ability to differentiate LiftPDF through browser-side/no-upload positioning;
- risk of cannibalization.

## 2. Priority Definitions

| Priority | Meaning | Publishing rule |
| --- | --- | --- |
| Priority 1 | High ROI, realistic long-tail, directly supports existing tools. | Publish first. Requires strong images and internal links. |
| Priority 2 | Useful and likely valuable, but either more competitive or less urgent. | Publish after Priority 1 cluster foundations. |
| Priority 3 | Good for long-term topical authority, but lower immediate ROI or riskier intent. | Publish only after Search Console validation. |

## 6. 12-Month Production Roadmap

### Month 1

Publish 4-6 pages:

- JPG to PDF without upload
- Merge PDF without upload
- PDF to JPG without upload
- Images to PDF without upload
- Multiple JPG to one PDF
- PNG transparency to PDF

Goal: establish the no-upload wedge.

### Month 2

Publish 4-6 pages:

- PDF page range to JPG
- One PDF page to JPG
- Combine PDFs in order
- JPG to PDF settings
- Combine JPG, PNG and WEBP into one PDF
- Extract text from PDF without upload

Goal: build practical depth around existing UI features.

### Month 3

Publish 4-6 pages:

- Password protect PDF locally
- Split PDF by page range
- Extract pages from PDF
- Why JPG gets blurry
- Why PDF to JPG downloads ZIP
- Why PNG has a white background

Goal: add troubleshooting and security trust.

### Months 4-6

Publish 12-18 pages from Priority 2:

- format comparisons;
- device tutorials;
- file size problems;
- image quality guides;
- organize PDF comparisons.

Goal: strengthen topical clusters and support broader category pages.

### Months 7-12

Publish selectively from Priority 3 based on Search Console:

- FAQ hubs;
- glossary pages;
- business/legal use cases;
- device pages;
- "best converter" pages only if authority is improving.

Goal: expand without creating thin content.

## 7. Cannibalization Control Map

| Query family | Primary page | Supporting pages | Avoid |
| --- | --- | --- | --- |
| jpg to pdf | `/jpg-to-pdf` | no-upload, multiple JPG, settings | Another generic "convert JPG to PDF" article. |
| png to pdf | `/png-to-pdf` | no-upload, transparency, multiple PNG | Rewriting JPG guide with PNG swapped in. |
| images to pdf | `/images-to-pdf` | mixed formats, any device, photos/receipts | Cannibalizing JPG/PNG pages. |
| pdf to jpg | `/pdf-to-jpg` | no-upload, one page, range, ZIP | A broad "PDF to image" page that competes. |
| merge pdf | `/merge-pdf` | no-upload, order, use cases | Generic "combine PDF online" duplicate. |

## 8. Internal Link Rules by Priority

### Priority 1 Pages

Must link to:

- parent tool;
- 2 sibling tools;
- 1 trust page;
- 1 category hub;
- 1 related guide if already published.

### Priority 2 Pages

Must link to:

- parent tool;
- 2-4 related guides;
- at least one comparison or problem page.

### Priority 3 Pages

Must link to:

- parent tool;
- closest FAQ hub;
- relevant category;
- one higher-priority page.

## 9. Production Batch Rules

Each publishing batch should include:

- 2 transactional guides;
- 1 problem guide;
- 1 comparison or use-case guide;
- no more than 1 device/platform guide.

This keeps the cluster balanced and avoids a wall of similar tutorials.

## 10. Validation Metrics

After each batch, check:

- indexed/not indexed status;
- impressions by query family;
- CTR;
- average position;
- tool page clicks from guide;
- scroll depth;
- CTA clicks;
- Search Console queries that deserve a new page;
- cannibalization between tool and guide.

## 11. Stop Conditions

Stop publishing a cluster temporarily if:

- pages are indexed but receive no impressions after 60-90 days;
- two pages compete for the same query;
- guide-to-tool clicks are near zero;
- content quality drops below the editorial checklist;
- product limitations make more content misleading.

## 12. Highest ROI Next Action

The next actual implementation phase should not create all planned pages. It should create:

1. guide template infrastructure;
2. `/guides` index;
3. metadata model for guide briefs;
4. image asset convention;
5. the first 3 guides only:
   - JPG to PDF without upload;
   - Merge PDF without upload;
   - PDF to JPG without upload.

Then wait for indexing and Search Console signals before scaling.
